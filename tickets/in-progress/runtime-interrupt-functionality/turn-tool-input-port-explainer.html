Why not put approval directly into ToolPhase?

  • The approval comes from outside: server/UI/backend.
  • It must enter the normal runtime boundary first: AgentMessageInbox.
  • Scheduler + handler validate turn ID and invocation ID.
  • Only valid approval wakes the private turn wait point.

Why have the private port?

  • Because ToolPhase may be waiting mid-turn.
  • The worker/inbox loop keeps running while ToolPhase waits.
  • The port wakes exactly the waiting invocation.
  • If the turn is interrupted, the port closes, so late approval cannot revive old work.
