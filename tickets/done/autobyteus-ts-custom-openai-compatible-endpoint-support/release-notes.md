## What's New
- You can now add reusable OpenAI-compatible providers from Settings using the visible `New Provider` row in the provider browser.
- Saved custom providers now appear alongside built-in providers and load their discovered models into the shared provider browser.

## Improvements
- Custom-provider models now show friendlier selector labels while built-in provider labels stay stable.

## Fixes
- Saved custom providers can now be deleted cleanly, with their models disappearing after refresh and staying gone after restart.
- Clicking the visible `New Provider` draft row still opens the full custom-provider editor flow.
